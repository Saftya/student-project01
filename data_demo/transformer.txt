The transformer is a neural network architecture designed to process sequences using attention instead of recurrence. Before transformers, many language models used recurrent neural networks that read tokens one at a time. That made long-range dependencies difficult and training slow. A transformer reads all tokens in a sequence at once and uses self-attention to decide which tokens should influence one another. This design made it possible to train much larger models on much larger datasets, and it became the foundation of modern language systems.

Self-attention works by creating query, key, and value vectors for each token. A token compares its query against the keys of other tokens to decide how much attention to pay to them. The values are then combined using those attention weights. Multi-head attention repeats this process several times in parallel, allowing the model to notice different kinds of relationships. One head might track subject-verb agreement, another might focus on references across a paragraph, and another might attend to punctuation or formatting clues.

Because self-attention alone does not know token order, transformers add positional information. Positional encoding gives the model a way to distinguish "the dog chased the cat" from "the cat chased the dog." The original transformer used sinusoidal positional encodings, while later models introduced learned or relative position methods. This small design choice is important because language depends heavily on order, and a bag of tokens is not the same as a sentence.

Transformer models are often grouped into encoder, decoder, and encoder-decoder families. BERT is an encoder-style model trained to build rich contextual representations of text. GPT is a decoder-style model trained to predict the next token, which makes it natural for generation. The original machine translation transformer used an encoder-decoder design: the encoder read the source sentence and the decoder produced the target sentence. These families share attention as a core mechanism but are optimized for different tasks.

BERT changed many NLP workflows because it produced contextual embeddings. The word "apple" can be represented differently in a sentence about fruit and a sentence about a technology company. GPT changed the public understanding of language models because next-token prediction at scale produced fluent generation, instruction following, summarization, and coding abilities. Both are transformer-based, but the way they are trained and used is different. This distinction helps students understand why an embedding model and a chat model are not interchangeable.

Transformers are powerful, but they are not databases. A model can generate an answer that sounds confident even when it does not have the needed facts in its parameters. Retrieval augmented generation solves part of this problem by giving the model relevant context at answer time. The retriever finds chunks from a trusted corpus, and the generator writes an answer using those chunks. This is why retrieval quality matters before generation quality. If the wrong context is retrieved, the transformer may produce a polished but incorrect answer.

In practical systems, transformers appear in several places. An embedding model may use a transformer encoder to convert text into vectors. A reranker may use a cross-encoder transformer to score a query-document pair more carefully than vector search can. A generator may use a decoder transformer to produce the final answer. A single RAG application can therefore contain multiple transformer models, each with a different job. Understanding the roles prevents students from treating "the model" as one mysterious black box.

The transformer architecture also has costs. Attention grows with sequence length, so very long contexts require more memory and computation. Engineers use chunking, retrieval, caching, quantization, and specialized attention methods to make systems practical. The tradeoff is always about information and cost: enough context to answer correctly, but not so much that the model becomes slow, expensive, or distracted by irrelevant text.
